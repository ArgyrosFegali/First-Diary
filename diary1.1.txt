今日はフリック先生にElixirとGitとGitHUBの使い方を習った。
Elixirはまだ導入したばかりなのでいまいちよくわかっていないが、
Kindleでいくつか本を見つけたので読み漁りながら触っていこうと思う。
そして肝心のGitについてだが手順を覚えるのに苦労しそうなので、
この日記をGitHUBにアップロードする手順について、
現状での自己解釈ではあるがここに記述しておこうと思う。
つまり私にとってはまだ予想と検証のステージだ。

１，変更内容の確認
git status
ディレクトリ内へのファイルの追加、
ファイルの内容が更新などを確認する。

２，ファイルの更新
git add .
変更があったファイルが間違いなければ、
コミット可能なように更新する。

３，コミットする
git commit -m "comment"
正直コミットするという動詞についてはいまいち理解できていないが、
コメントをつけるらしいので多分タグ付けか何かをしている印象がある。

４，プッシュする
git push -f
要するにGitHUBへアップロードする。
-fのパラメータは強制的にという意味らしい。

というわけで以上が一日目である。
ひとまず頑張って明日も日記を上げてみようかとは思っている。
